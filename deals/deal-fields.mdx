## Deal Fields Overview

Deal fields allow you to track additional information about deals beyond the basic title and value. You can also customize your pipeline stages.

## Custom Fields for Deals

### Field Types

Similar to lead custom fields, deal fields support:
- **Text**: Single-line text
- **Textarea**: Multi-line text
- **Number**: Numeric values
- **Date**: Date values
- **Dropdown**: Single selection
- **Multiselect**: Multiple selections
- **Boolean**: Yes/No values

### Creating Deal Fields

1. Navigate to **Settings** → **Deal Fields**
2. Click **"Add Field"** or **"Create Field"**
3. Configure field:
   - Field name (internal)
   - Field label (display)
   - Field type
   - Description
   - Required status
4. Save field

### Using Deal Fields

Deal fields appear in:
- Deal creation form
- Deal edit form
- Deal details sidebar
- Pipeline cards (key fields)

## Pipeline Stages

### Default Stages

Your pipeline starts with default stages:
- New/Qualification
- Proposal
- Negotiation
- Closed Won
- Closed Lost

### Customizing Stages

**Add Stages:**
1. Go to **Settings** → **Deal Pipeline**
2. Click **"Add Stage"**
3. Enter stage name
4. Choose color
5. Set stage order
6. Save

**Edit Stages:**
- Modify stage name
- Change stage color
- Reorder stages
- Set default stage

**Remove Stages:**
- Delete custom stages
- Cannot delete required stages (Won/Lost)
- Deals in deleted stage move to default

### Stage Configuration

**Stage Properties:**
- **Name**: Display name
- **Color**: Visual identifier
- **Order**: Position in pipeline
- **Default**: Default stage for new deals

## Best Practices

1. **Relevant Fields**: Create fields that add value
2. **Stage Names**: Use clear, descriptive stage names
3. **Stage Colors**: Use colors to distinguish stages
4. **Logical Order**: Order stages to match your process
5. **Keep It Simple**: Don't create too many stages or fields
